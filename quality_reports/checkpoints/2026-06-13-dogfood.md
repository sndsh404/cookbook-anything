## What the run proved, and the bugs it caught

The point was to prove the pipeline, not to write a nice README. Three real
defects surfaced and were fixed, which is the proof:

1. **Broken marker.** The writer quoted DESIGN.md's own `{{c:0317}}` syntax
   example verbatim into the claims appendix, and the verifier parsed it as a
   real marker. Fixed: claim extraction rejects sentences containing `{{`/`}}`
   (same class as the `[REDACTED` rule). A gate caught this; the run failed
   and was re-run, per the rules.
2. **Architecture figure misrepresented the repo.** Extraction was
   Python-only, so the model contained figlib + docs but NOT the Rust core or
   TS runner, and the page-one figure showed the project as if it were a
   Python+docs codebase. Fixed by adding real line-based Rust and TS
   extractors (`core/ca-extract/src/nativecode.rs`); the figure now shows the
   true core (rust) / figlib (python) / docs split. This was the honest
   decision point flagged before the run: option (a), extend extraction, was
   tractable because intake already ingests .rs/.ts spans.
3. **Referenced-but-missing figures.** Markdown-only chapters referenced a
   chapter figure the figure stage never produced (markdown file nodes lacked
   a `loc` attr, so the quantity recipe skipped them). Fixed two ways: doc
   file nodes now carry `loc`, and a new figcheck rule (F-15) flags any prose
   reference to a figure that was not rendered. No gate caught this one before;
   now one does.

Also fixed the bare-module import relink (`from render import x` now resolves
to figlib/render.py), which turned figlib's real internal dependencies into a
genuine dependency_graph figure (every recipe imports the style.py hub).

## Honesty stated in the README (rule 6)

- Extraction covers Python/Rust/TS source + Markdown structure; config files
  are ingested as spans but not graphed.
- The page-one figure is a 3-of-6-cluster summary, not the full graph.
- The stage pipeline is orchestrated in stages.ts, not extracted as a
  dataflow graph; figures show real code structure, not a stage diagram.
- The LLM judgment layers (vision-critic, humanize-auditor) are not built;
  the gates shown are the deterministic ones.
