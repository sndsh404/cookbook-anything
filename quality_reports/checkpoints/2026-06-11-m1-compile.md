## What shipped

- `scripts/model_schema.py` hand-rolled validator for the DESIGN 4 contract.
- `scripts/extract_code.py` python ast extractor: defines/contains/imports
  always, same-file calls only (narrow-and-true).
- `scripts/extract_doc.py` markdown sections, verbatim-sentence claims,
  bold-term glossary.
- `scripts/extract_data.py` schema-first CSV/SQL (tables, columns, FKs; no
  row values).
- `scripts/merge.py` dedupe/dangler-drop/span-prune + the agent-edge
  confidence clamp (cap 0.75).
- `scripts/compile_model.py` orchestrator: extract -> link imports ->
  deterministic glossary from class docstrings -> merge -> validate.

## Decisions

- Extractors read the REDACTED span store, never raw sources, so secret
  hygiene survives compilation by construction.
- Claims are sentences lifted verbatim from doc paragraphs: the claim text IS
  span text. No paraphrase at compile time.
- Agent edge clamp lives in merge.py (cap 0.75), and model_schema rejects any
  agent edge at 1.0, so the rule is enforced twice.
- The entropy filter redacted 1316 strings in llmwiki (mostly lockfile hashes
  and minified blobs). False positives redact non-secrets, which fails safe;
  noted as acceptable for v0.
