## What shipped (figlib/, python leaf renderer)

- `style.py` house style: semantic closed palette (ink/accent/good/warn/
  muted/paper + 4 fills), print + sketch modes, default cycle banned.
- `payload.py` the figure data contract: node IDs, edge refs, span-backed
  quantities, layout hints. Nothing else can reach a recipe.
- `introspect.py` facts from the live artist tree (colors, font sizes, label
  bboxes, legends, titles) written to a sidecar. Ground truth = what was
  actually drawn.
- `recipes/` architecture_box, dataflow, dependency_graph, pipeline_stages,
  annotated_code, quantity. Density ceilings enforced; recipes refuse to
  cram (F-09 raises).
- `render.py` declares the Figure Read before drawing, embeds payload sha +
  node IDs in PNG metadata, writes the sidecar.
- `figcheck.py` the rule catalog F-01..F-14 as mechanical checks over
  sidecar + model.json; findings cite rule IDs.
- `seeded_defects/generate.py` ten planted violations + toy model.

## Decisions

- v0 critic = figcheck's deterministic battery. The LLM vision critic
  (judging the PNG against its declared read) is the judgment layer on top
  and is NOT yet implemented; the seeded-defect gate is built so it can be
  pointed at that critic unchanged. Recorded honestly per DESIGN seam 1.
- Grid color #dddddd added to the allowed palette (it is house style).

## The loop caught real bugs

First run on real data: F-03 (annotation column overlapping long code

catalog does not yet encode (vertical arrows exiting boxes sideways,
